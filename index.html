O que será introduzido no Módulo 3

Github

No Capítulo 18, você aprenderá a hospedar seus exercícios e projetos usando ferramentas gratuitas como GitHub Desktop e GitHub Pages. O objetivo é permitir que você sincronize todo o seu código entre casa, escola, faculdade ou qualquer outro computador — tudo usando a nuvem. Esse capítulo é especialmente útil para quem faz o curso em ambientes acadêmicos ou trabalha entre dois dispositivos diferentes.

No Capítulo 19, o foco é totalmente visual: você vai dominar as propriedades de background no CSS, aprendendo a trabalhar com imagens de fundo, centralização, tamanho, posicionamento, efeitos estéticos e até o famoso parallax, muito usado em sites modernos. O professor explica desde os conceitos individuais de cada propriedade até o uso de shorthands, tornando seu código mais eficiente.

Com essa base, você avança para o Capítulo 20, onde criará o primeiro grande projeto do módulo: o Projeto Cordel. Inspirado em um cordel de Milton Duarte, esse projeto combina textos, imagens e efeitos parallax em uma página totalmente responsiva, pensada especialmente para dispositivos móveis. O professor mostra como montar tudo do zero, ajustar estilos para telas pequenas e grandes e preparar o site para a web.

Ainda no Capítulo 20, você também aprende a hospedar o projeto gratuitamente, permitindo compartilhar seu site com qualquer pessoa e acessá-lo em qualquer dispositivo. O professor demonstra como publicar a página e como garantir que ela continue responsiva mesmo em telas maiores como notebooks e monitores.

Por fim, no Capítulo 21, você mergulha no mundo das tabelas em HTML5 — não para criar layouts como no passado, mas para tabular dados de forma moderna, eficiente e profissional. Você aprenderá formatação básica, alinhamento de conteúdo, mesclagem de células, tabelas responsivas, cabeçalhos fixos, tratamento de tabelas grandes e até o famoso efeito zebrado, que melhora a leitura das linhas. Tudo usando HTML5 semântico e CSS moderno.

Capítulo 18

O que é Git e GitHub?

Nesta aula, iniciamos oficialmente os estudos do Capítulo 18, onde o foco deixa temporariamente o HTML e o CSS para abrir espaço a um tema essencial para qualquer desenvolvedor moderno: o versionamento de software. Depois de concluir os 17 primeiros capítulos e até criar um mini-projeto prático — aquele site que apresentava a história do mascote Android — agora é hora de aprender a proteger, organizar, evoluir e publicar seus códigos de forma segura.

O professor começa explicando por que o versionamento é tão importante. Você vai entender os conceitos básicos de Git e GitHub, duas ferramentas poderosas que permitem manter versões do seu projeto, recuperar estados anteriores caso algo dê errado, trabalhar em equipe e até hospedar seus exercícios gratuitamente. Apesar de serem ferramentas avançadas, a abordagem aqui é introdutória, mostrando exatamente o que você precisa para publicar seus projetos e manter cópias seguras na nuvem — sem complexidade desnecessária.

Nessa aula é explicado que o Github é uma página utilizada para repositórios de códigos, onde programadores do mundo inteiro usam e compartilham seus projetos, você pode estudar, clonar e modificar repositórios, também acessar seus códigos em qualquer dispositivo clonando-os, seja do pc do amigo, no celular, no computador da faculdade, você escolhe.

Também é apresentado o Git, programa onde você salva seus projetos ao longo de atualizações e correções que você faz no seu código, você pode dar "commit", explicando o que foi mudado no código e quando foi mudado para evitar perdas e erros caso se você queira mudar para uma versão anterior do seu código, ele é uma ótima opção, combado com o Github Desktop, você pode dar "push" para subir seus arquivos para seu repositório do GitHub.

Na aula, também praticamos o uso do Github Desktop, criando repositórios e manuseando-o.

Requisitos:

Para trabalhar no seu projeto em outro dispositivo, como requisito você terá que instalar: Github Desktop, Visual Studio Code, Navegador & Git.

Como trabalhar em outro dispositivo:

Github Desktop > File > Options > Accounts > Github.com Sign in.

Neste momento o Github estará funcionando após seu login, entretanto o Git não estará, portanto siga os passos abaixo.

File > Options > Git > Clique em salvar.

Clique no repositório desejado > Clonar > Crie uma pasta > Clonar. Pronto, você já tem seu projeto para trabalhar em outro dispositivo!

Como remover sem deixar rastros

1 - Repositório > Remove > Opção lixeira > Remove.
2 - File > Options > Sign Out > Git > Apague os campos da sua conta.

Comandos para personalizar
background-size: 100px 100px; #Tamanho do fundo
background-repeat: no-repeat; #A imagem repete?

importante:

Para uma abordagem exata do posicionamento do seu site use ao invés de height: 100px; use height: 100vh; 'vh' significa viewport, é aonde a sua página estará ancorada no redimensionamento da sua página.

No posicionamento do background o comando é: background-position: center bottom;, primeiro vem a coluna: left, center, right, e em seguida o lugar, top, center, bottom.

dimensionamento

existem alguns comandos para dimensionar a imagem melhor no fundo, dentre alguns está, background-size: 100%;

background-size: 100% 100% - estica a imagem para caber independente do dispositivo, pode ficar esticada
background-size: contain; - mostra 100% a imagem exibida
background-size: cover; - ocupa 100% da viewport, sem esticar, perfeito

shorthands

O uso de shorthands são linhas de códigos pequenas ao invés de grandes, preservando espaço no workspace.
color > image > position > repeat > size > attachment
Por algum motivo o size não funciona no HTML5 em shorthands, por isso use-o separado.

Anatomia das tabelas grandes

THEAD TR,TD,TH
TBODY TR,TD,TH
TFOOT TR,TD,TH

tr (Table Row): Representa uma linha horizontal completa dentro da tabela. Você usa uma para cada nova linha que deseja criar.

th (Table Header): Uma célula de cabeçalho. O texto geralmente aparece em negrito e centralizado por padrão. É usada dentro do thead (ou no início de uma linha) para indicar o título daquela coluna/linha.

td(Table Data): Uma célula de dado comum. É onde você insere a informação propriamente dita. Fica sempre dentro de uma tr.
